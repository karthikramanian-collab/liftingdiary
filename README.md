# LiftingDiary

A personal workout tracking app for serious lifters. Log every set, track progressive overload, and build a permanent record of your training — one session at a time.

---

## What it does

- Log workouts with exercises, sets, reps, weight, and RPE
- Track warmup vs working sets
- View training history per user
- Industrial-themed UI built for focus, not fluff

## Database Schema

Four tables with cascade deletes on the workout → sets path:

```
exercises
  id, name, muscle_group, category, created_by, created_at

workouts
  id, user_id (Clerk), name, notes, started_at, finished_at, created_at

workout_exercises  (junction: workout ↔ exercise)
  id, workout_id → workouts, exercise_id → exercises, order_index, notes, created_at

sets
  id, workout_exercise_id → workout_exercises
  set_number, reps, weight_kg, duration_seconds, rpe, is_warmup, notes, created_at
```

Drizzle relations are defined in `src/app/db/schema.ts` for typed query building.

## Getting Started

### Prerequisites

- Node.js 20+
- A [Supabase](https://supabase.com) project (PostgreSQL)
- A [Clerk](https://clerk.com) application

### Environment variables

Copy `.env.example` to `.env` and fill in:

```bash
DATABASE_URL=postgresql://...   # Supabase connection string
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_...
CLERK_SECRET_KEY=sk_...
```

### Install and run

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

### Database migrations

```bash
# Generate a migration from schema changes
npx drizzle-kit generate

# Push migrations to the database
npx drizzle-kit migrate
```
